    Class for doing database lookups, The look up is done Lazily i.e. Lazy Loading.

    This class provides several methods for interacting with the database with one important thing to note is that some methods return a Queryset object while other return the expected result object.

    package powerorm\queries
    since 1.0.0
    author Eddilbert Macharia (http://eddmash.com)

    filter(array $conditions) : \powerorm\queries\Queryset

    Implements the where part of a query and returns rows limited by the where clause.

    USAGE:

    Normal Filtering

    To fetch all rows in the database with the name john.

    $this->User_model->filter(array('name'=>'john'))

    To fetch all rows in the database with the name=john and age=20.

    $this->User_model->filter(array('name'=>'john', 'age'=20))

    Related Filter

    To fetch all rows based on there related data e.g fetch user model where role is admin.

    
 $roles = $this->user_model->filter(array("role::name"=>"admin"));

    Be More Specific

    You can be more specifics on your conditions by using the lookup options available to you. $this->user_model->filter('username__contains'=>'eri');

    this will run the following sql select * from user_table where username like %eri%

    see To fetch only one item.

    Parameters

    $conditions

    array

    the where condition e.g. array('username'=>'john', 'password'="ad+as')

    get(array $values) : array | boolean | object

    This method Returns a Queryset for further refinement USAGE:.

    To fetch a single row in the database with the name john.

    $this->User_model->get(array('name'=>'john'))

    To fetch a single row in the database with the name=john and age=20.

    $this->User_model->get(array('name'=>'john', 'age'=20))

    To fetch a single row in the database with the primary key 1, where primary key column is id.

    $this->User_model->get(1)

    if primary key is not id you can pass the primary key field

    $this->User_model->get(array('pk'=>1))

    see To fetch more than one item based on some conditions.

    Parameters

    an array of field and the value to look for, or an integer-primary key.
